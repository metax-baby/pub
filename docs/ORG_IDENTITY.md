The organization is treated as a **dev command center**: a public/private GitHub working area for development control, public surfaces, low-sensitivity reference material, and coordination scaffolds.

It is not ZERO. It is not the permanent root of PUNNARAJ. It is a provider-hosted operational surface that can be mirrored, rebuilt, replaced, or migrated.

## Role

The dev command center may contain:

- public shared-understanding pages
- public or private development repositories
- MCP/server/client experiments
- GitHub Pages or Cloudflare Pages source repositories
- low-sensitivity doctrine snapshots
- operating-model documents
- coordination notes for agents and humans

## Boundary

Do not store raw secrets here.

Do not treat GitHub organization state as final identity truth.

Do not duplicate provider ledgers that GitHub already maintains. Instead, reference GitHub history and add missing PUNNARAJ meaning layers such as role, relation, sensitivity, cost, risk, and recovery path.

## Recovery stance

This organization is useful while GitHub is useful.

PUNNARAJ should keep enough mirrored repository data and metadata in controlled storage so that the same working structure can be rebuilt on another provider or self-hosted service if needed.
